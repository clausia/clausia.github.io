bool cfpmath::parity_order_inversion_base_2 ( const int  number_in)

This function returns the number of 1,1 swaps required to invert a number

Consider a number 29 =0 1 1 1 0 1 a b c d e f

If we want to invert the cables, we would obtain a number of intersections: 

                  f e d c b a
                  1 0 1 1 1 0

So there is a number of crossings. The question is how many ones appear in the crossing? For example, consider 4 bits:

0 0 0 0 => Require no crossing (Even parity) 0 0 0 1 => Require no crossing (Even parity) 0 0 1 0 => Again no crossing (Even parity) 0 0 1 1 => Require one crossing(Odd parity)
